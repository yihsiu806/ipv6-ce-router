NAME

  Host Ignores Router Solicitations 
  Part A: All-Router Multicast Destination

Purpose

   Verify that a host ignores Router Solicitations and does not update its Neighbor Cache.

References

[RFC 7084],W-3 [RFC 4861]- Section 6.2.6

Resource Requirements

  * Packet generator
  * Monitor to capture packets

Test Setup

  Common Test Setup 1.2 is performed at the beginning of this test part.
  The Common Test Cleanup procedure is performed after each part.
Router Solicitation A
IPv6 Header
Next Header: 58
Destination Address: [See below]
Router Solicitation
Source Link-layer Address Option

Procedure

  Part A: All-Router Multicast Destination
   1. TN1 transmits Router Solicitation A. The Destination Address is the All-Router multicast
      Address.
   2. Wait (RETRANS_TIMER * MAX_*CAST_SOLICIT). (3 seconds)
   3. TN1 transmits a link-local Echo Request to the CE-Router.
   4. Wait 2 seconds.
   5. Observe the packets transmitted by the CE-Router.

Observable Results

  Parts A-C
   In all Parts, the CE-Router should send a multicast Neighbor Solicitation for TN1 in Step 5,
   indicating the CE-Router did not process the Router Solicitation from TN1.

Possible Problems

  * None.

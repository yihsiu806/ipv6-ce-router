NAME

  Neighbor Solicitation Processing, NCE State INCOMPLETE
  Part C: Unicast Neighbor Solicitation without SLL

Purpose

  Verify that a node properly updates its neighbor cache upon receipt of neighbor solicitations
  when the NCE of the neighbor is in state INCOMPLETE.

References

[RFC 7084],L-1 [RFC 4861],Section 7.2.3 and 7.2.4

Resource Requirements

  * Packet generator
  * Monitor to capture packets

Test Setup

  Common Test Setup is performed at the beginning of this test part.
Packet A
IPv6 Header
Next Header: 58
Source Address: TN2's Link-local Address
Destination Address: NUT's Link-local Address
ICMPv6 Echo Request

Neighbor Solicitation B
IPv6 Header
Next Header: 58
Destination Address: NUT's Link-local Address
Source Address: TN2's Link-local Address
Neighbor Solicitation
Target Address: NUT's Link-local Address
Source Link-Layer Address: TN2's Ethernet address
Neighbor Solicitation C
IPv6 Header
Next Header: 58
Destination Address: NUT's Solicited-node Multicast Link-local Address
Source Address: TN2's Link-local Address
Neighbor Solicitation
Target Address: NUT's Link-local Address
Source Link-Layer Address: TN2's Ethernet address

Procedure

  Part C: Unicast Neighbor Solicitation without SLL
   13. TN2 transmits Echo Request A.
   14. Check the NCE of TN2 on the NUT and observe packets transmitted by the NUT.
   15. TN2 transmits Neighbor Solicitation B without the Source Link-layer Address option.
   16. Check the NCE of TN2 on the NUT and observe packets transmitted by the NUT.

Observable Results

  Part C

   Step 14:
   The NUT should create a Neighbor Cache Entry for TN2 and set the state of the
   Entry to INCOMPLETE. The NUT should send a multicast Neighbor Solicitation to TN2.

   Step 16: 
   After receiving TN2's Neighbor Solicitation, the NUT should not update the NCE

Possible Problems

  * None.

NAME

  Invalid Neighbor Solicitation Handling
  Part A: Invalid Target Address

Purpose

  Verify that a node takes the proper actions upon receipt of an invalid Neighbor Solicitation.

References

[RFC 7084],L-1 [RFC 4861],Section 7.3

Resource Requirements

  * Packet generator
  * Monitor to capture packets

Test Setup

  Common Test Setup is performed at the beginning of this test part.
Neighbor Sol. A

Hop Limit: 255

Souce Link-layer: Address: TN2's Link-layer address

Hop Limit: 255
Neighbor Sol.
Target Address: NUT's Link-local Address
Source Link-layer Address: TN2's Link-layer address

Procedure

  Part A: Invalid Target Address
   1. TN2 transmits Neighbor Solicitation A with the Target Address set to the CE-Routers Multicast.
   2. Observe the packets transmitted by the NUT.

Observable Results

  Part A
   Step 2: The NUT must not transmit any packets corresponding to Neighbor Solicitation A.

Possible Problems

  * None.
